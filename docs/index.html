Jsonotron

Jsonotron is...

  • a small set of components for building a **NodeJS microservice**
  • for storing, patching and querying **JSON documents**
  • stored in a schemaless/NoSQL **database**
  • that have known, enforceable, and **evolving schemas**.

NoSQL Database + Jsonotron = A microservice for your curated/governed data.

A Jsonotron setup is based on 3 parts: a backend document store, the jsonotron core and a front-end interface.

Backend Data Stores:

  • jsonotron-memdocstore: An in-memory document store.
  • jsonotron-cosmosdb: A connector for an Azure Cosmos DB document store.

Jsonotron Core:

  • jsonotron: The core library that reads schemas and manages CRUD and custom operations.

Front-end Interfaces:

  • jsonotron-express: An ExpressJS middleware that routes RESTful requests into the system.
